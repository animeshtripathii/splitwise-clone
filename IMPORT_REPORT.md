# Import Report

This report documents the ingestion and cleaning of `Expenses_Export.csv` into the Splitwise Clone database as per the assignment requirements.

---

## 1. Cleaned Anomaly Resolution Log

During the import, the application automatically parsed, standardized, and cleaned the raw CSV data according to the following engineering decisions:

| Row | Issue / Anomaly | Resolution / Action Taken |
| :--- | :--- | :--- |
| **5 & 6** | **Duplicate Entry**: Marina Bites dinner logged twice by Dev. | Imported the first entry (proper casing + notes) and skipped the second. |
| **7** | **Formatted Amount**: Electricity Feb amount was `"1,200"` (with comma). | Parsed string to remove commas and converted to `Decimal('1200.00')`. |
| **10** | **Excess Decimal Places**: Cylinder refill amount was `899.995`. | Rounded to 2 decimal places: `Decimal('900.00')`. |
| **11** | **Name Alias**: Payer was listed as `Priya S`. | Mapped `Priya S` to the existing user `Priya` (email `priya@example.com`). |
| **13** | **Missing Payer**: House cleaning supplies payer field was empty. | Defaulted payer to `Aisha` (group creator) to keep the balance sheet intact. |
| **14** | **Settlement in Expense Log**: `Rohan paid Aisha back` 5,000 INR. | Logged directly into the database as a `Settlement` between Rohan and Aisha, rather than as an expense. |
| **15 & 32** | **Unnormalized Percentages**: Split percentages summed to 110%. | Normalized percentages (divided each share by 1.10) and distributed the rounding remainder. |
| **20, 21, 23, 26** | **USD Currency**: Expenses logged in USD (`Goa villa booking`, etc.). | Converted USD to INR using a fixed exchange rate of **83.00 INR/USD**. |
| **23** | **External Guest**: `Dev's friend Kabir` was listed in the split. | Registered a new account for Kabir (`kabir@example.com`) and temporarily added him to "The Flat" group to accurately calculate split shares. |
| **24 & 25** | **Double Logged Event**: Thalassa dinner logged by Aisha (2,400) and Rohan (2,450). | Imported Rohan's entry (2,450) and skipped Aisha's (2,400) based on notes flagging Aisha's entry as incorrect. |
| **27** | **Mixed Case/Spaces & Date Format**: `Mar-14` date and trailing spaces on name `rohan `. | Parsed date as `2026-03-14` and mapped name to standard `Rohan`. |
| **28** | **Missing Currency**: Groceries DMart had a blank currency field. | Defaulted currency to `INR`. |
| **31** | **Zero-Amount / Correction**: Swiggy dinner had amount `0`. | Skipped this entry entirely as it was flagged as a duplicate/correction. |
| **34** | **Date Out-of-Order**: Date was `04-05-2026` but sat chronologically between March 28 and April 1. | Parsed as `2026-04-05` (April 5) matching the chronological context of the rows. |
| **38** | **Settlement in Expense Log 2**: `Sam deposit share` 15,000 INR to Aisha. | Logged as a `Settlement` from Sam to Aisha, rather than as an expense. |
| **42** | **Redundant split details**: Split type was `equal` but listed redundant share weight details `Aisha 1; Rohan 1; ...` | Treated as a standard equal split among the participants listed. |
